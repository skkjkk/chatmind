# ChatMind - 聊天分析与回复助手

## 项目概述

一个帮助用户分析微信聊天记录、了解对方性格习惯，并提供智能回复建议的私密AI应用。

## 二、完整功能列表

### 2.1 数据分析功能

#### 消息基础统计

| 功能 | 说明 |
|------|------|
| 消息总数 | 统计聊天记录总消息数 |
| 消息占比 | 双方各自发送消息数量及占比 |
| 消息类型分布 | 文字/语音/图片/表情/文件 各占比例 |
| 消息长度统计 | 平均消息字数、最长/最短消息 |

#### 时间维度分析

| 功能 | 说明 |
|------|------|
| 活跃时间段 | 统计双方各时段（凌晨/早晨/上午/下午/晚上）发送消息频率 |
| 熬夜指数 | 统计凌晨0-5点发送消息的频率 |
| 工作日vs周末 | 对比工作日和休息日聊天活跃度 |
| 关系时间线 | 整体时间跨度、每日消息趋势 |

#### 词汇与表达分析

| 功能 | 说明 |
|------|------|
| 高频词汇TOP50 | 提取双方各自最常用的词汇 |
| 词云展示 | 可视化高频词汇 |
| 语气词统计 | 哈哈、嗯嗯、哇、啊等语气词使用频率 |
| 常用表情包 | 统计双方最常用的emoji/表情包TOP10 |
| 标点使用习惯 | 是否喜欢用 "！！"、"。。。" 等 |

#### 性格分析（核心）

| 维度 | 分析内容 | 评估方式 |
|------|----------|----------|
| 外向/内向 | 谁更主动发起对话、消息数量对比 | 消息数量、发起对话频率 |
| 理性/感性 | 讲道理多还是谈感情、情绪词汇占比 | 关键词分类统计 |
| 直接/委婉 | 说话直来直去还是绕弯子 | 表达方式分析 |
| 积极/消极 | 整体情绪倾向正面还是负面 | 情感分析模型 |
| 依赖度 | 谁更依赖这段关系、回复是否及时 | 响应时间、消息频率 |
| 表达方式 | 文字派还是表情包派 | 消息类型占比 |
| 互动模式 | 有问必答 / 爱答不理 / 互相调侃 / 单方面付出 | 互动模式识别算法 |

#### 关系分析

| 功能 | 说明 |
|------|------|
| 亲密度评分 | 基于消息频率/内容亲密度综合打分 0-100 |
| 主动指数 | 统计发起对话的频率，双方对比 |
| 关系变化趋势 | 对比早期vs现在，关系升温/降温/平稳 |
| 角色分析 | 谁是话题发起者、谁更被动回应 |

### 2.2 回复建议功能

#### 模式一：智能语境模式

```
输入内容：
- 你想回复的话（草稿）
- 当前对话背景（对方说了什么）

输出内容：
- 语境分析（对方情绪、意图）
- 3个回复示例（正式/俏皮/简洁/热情风格）
- 优化建议（帮你改得更好）
- 续写完整回复
```

#### 模式二：快速问答模式

```
输入内容：
- 场景描述（如：对方问我借5000块钱）
- 或直接问题（如：怎么优雅地要债）

输出内容：
- 场景分析
- 3-5个回复建议
- 不同风格的回复示例
```

#### 回复风格选择

| 风格 | 适用场景 |
|------|----------|
| 正式 | 商务沟通、不太熟的人 |
| 俏皮 | 关系好、轻松氛围 |
| 简洁 | 对方话少、快速回复 |
| 热情 | 想拉近关系、表达友好 |
| 委婉 | 拒绝、不好意思的情况 |

## 四、微信聊天记录格式支持

### 支持的导出格式

| 格式 | 来源工具 | 解析难度 |
|------|----------|----------|
| HTML | 微信PC版备份/第三方导出工具 | 中等 |
| JSON | 部分导出工具 | 简单 |
| TXT | 手动导出 | 简单 |

### 解析字段

解析后需要提取的核心字段：

```
- sender: 发送者名称
- content: 消息内容
- timestamp: 发送时间
- message_type: 消息类型（text/image/voice/emoji/file）
- is_from_me: 是否为自己的消息
```

## 五、部署方案

### 开发环境

```bash
# 后端
cd backend
pip install -r requirements.txt
python -m uvicorn app.main:app --reload --port 8000

# 前端
cd frontend
npm install
npm run dev
```

### 生产部署

| 步骤 | 操作 |
|------|------|
| 1. Git推送 | 代码推送到GitHub |
| 2. 连接Railway/Zeabur | 关联GitHub仓库 |
| 3. 配置环境变量 | DEEPSEEK_API_KEY等 |
| 4. 部署 | 自动构建和部署 |
| 5. 域名绑定 | 可选自定义域名 |

### 环境变量

```env
# 后端
DEEPSEEK_API_KEY=sk-xxxx
JWT_SECRET_KEY=your-secret-key
DATABASE_URL=postgresql://...
CORS_ORIGINS=https://your-domain.com

# 前端
VITE_API_URL=https://your-backend-url
```
